# Carbo — 3-minute demo

> Click-by-click walkthrough. Each step quotes what a reviewer will actually see, so this doc is gradeable without running the app.

## Setup (one command)

```bash
nvm use            # Node 22 LTS — do NOT run on Node 25
npm install
npm run reset      # wipe + migrate + seed (61 sandbox tx over 90 days)
npm run dev        # http://localhost:3000
```

Defaults are safe: `ANTHROPIC_MOCK=1`, `BUNQ_MOCK=1`, `DRY_RUN=1`. No external keys needed for the demo flow.

## The walk

### 1. `/` — Dashboard (~20 s)

Hero card: reserve balance and offset rate. KPI strip: **61 transactions**, **€39k spend** over 90 days, **CO₂e estimate with confidence bar**, projected tax savings, avoidable impact. 6-month spend × CO₂e dual-axis chart. 5 category spend bars (fuel, electricity, goods, travel, digital).

The **Run Carbon Close** button is the entry to the close state machine.

> Innovation cue: every CO₂e number ships with a `ConfidenceBar`. No bare numbers.

### 2. Click **Run Carbon Close** → `/close/[id]` (~30 s)

The pipeline animates through the close state machine:

```
INGEST → CLASSIFY → ESTIMATE_INITIAL → CLUSTER_UNCERTAINTY → QUESTIONS_GENERATED
```

The cluster step picks the 2–3 highest-impact uncertain clusters scored by `spend × (1 − confidence)`. **Only those generate refinement questions** — the rest stay deterministic.

> Technical-execution cue: state column on `close_runs` advances live; you can `SELECT * FROM close_runs` mid-run and see the row pinned at the current state.

### 3. Answer the 3 refinement questions (~30 s)

Each question is a multiple-choice on a real transaction cluster (e.g. "Was this Shell purchase petrol, diesel, or EV charging?"). Answering re-classifies the merchant cache and reruns the emissions estimate for the cluster. **Confidence rises** as `method` flips from `spend_based` to `refined`.

> Creativity cue: the agent doesn't ask 60 questions. Spend-weighted clustering means 2–3 high-leverage answers cover ~80% of the uncertainty.

### 4. Review proposed actions, click **Approve & transfer €X** (~20 s)

The proposed actions: an **intra-user reserve transfer** (main account → Carbo Reserve sub-account) plus a recommended EU carbon-credit allocation across biochar / peatland / reforestation totaling ~4.8 t. The amount comes from the policy evaluator (`lib/policy/evaluate.ts`), which is pure / auditable / deterministic.

Click **Approve & execute**. State machine transitions: `EXECUTING → COMPLETED`. With `DRY_RUN=1`, the audit log gets `action.reserve_transfer` but no money moves. With `DRY_RUN=0` and `BUNQ_MOCK=0`, the bunq sandbox transfer fires.

> bunq cue: this is a real `intraUserTransfer` call (`lib/bunq/payments.ts` → `lib/agent/close.ts:442`). Signed RSA-SHA256.

### 5. `/report/2026-04` — CSRD report (~20 s)

ESRS E1-6 (Scope 1/2/3 totals with confidence ranges) and E1-7 (carbon credit allocation) tables. Methodology footnote cites factor sources (DEFRA 2024 / ADEME / Exiobase). For live mode, the prose is generated by Sonnet 4.6 (`lib/agent/narrative.ts`); in mock mode it's a deterministic template.

### 6. `/ledger` — Audit chain (~15 s)

The hash chain renders one row per audit event. SHA-256 link to previous row. **Chain-valid badge** confirmed via `verifyChain` (`lib/audit/append.ts:47`). UPDATE/DELETE on `audit_events` is blocked by SQL trigger; tampering breaks the chain on next verify.

### 7. `/impacts` — Impact matrix (~25 s)

The **2×2 matrix** (cost vs. carbon) classifies recommendations into Quick wins / Green investments / Cost savers / Avoid. Top 5 switches with before/after bars. Industry benchmark bar (you vs. peers from Exiobase). Client-side **What-if simulator** to toggle multiple switches and see cumulative Δ cost + Δ CO₂e.

### 8. `/presentation` — Interactive deck (~20 s)

7-section scroll-synced deck that *replays* the 8-agent DAG with evidence chips and color-coded agent groups. This is the **innovation showcase** — it shows the DAG as a first-class artifact, not a hidden internal.

## Optional — full live LLM run

```bash
ANTHROPIC_MOCK=0 ANTHROPIC_API_KEY=sk-... npm run dev
```

Now refinement questions, narrative, judge verdicts, and recommendations are all real Sonnet 4.6 output. The Research agent uses Anthropic's native `web_search_20250305` tool (with a 30-day `research_cache` table). Token / cache-hit-rate instrumentation populates the `agentRuns` and `agentMessages` tables.

`npx tsx scripts/dag-smoke.ts` runs the full 8-agent DAG in one shot for a one-screen sanity check.

## Optional — live bunq sandbox

```bash
npm run bunq:keygen           # generates RSA keypair
npm run bunq:sandbox-user     # mints sandbox user + API key
BUNQ_MOCK=0 npm run bunq:bootstrap        # 3-leg auth, persists session
BUNQ_MOCK=0 npm run bunq:create-reserve   # creates "Carbo Reserve" sub-account
BUNQ_MOCK=0 npm run bunq:sugardaddy       # seeds €500 from sandbox faucet
npm run dev:live              # Cloudflare tunnel + dev server, prints public URL
BUNQ_MOCK=0 BUNQ_WEBHOOK_URL=<tunnel> npm run bunq:register
```

Then make a real sandbox payment from another bunq account → the webhook lands → the transaction appears in the dashboard within a second.

## What a reviewer should walk away seeing

1. A 2–3 question monthly close that takes a busy founder under a minute.
2. A real bunq Reserve transfer, signed, audit-logged, dry-run-safe by default.
3. An agent DAG where the LLM writes prose but **code adjudicates the math and the money**.
4. A CSRD report and a hash-chained ledger that an external auditor would accept.
